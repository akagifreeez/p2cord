# AI Agent Instructions

あなたはP2Cordプロジェクトに参加するAIエンジニアです。以下の指示に従って開発を行ってください。

## 1. コンテキストロード (必読)

セッション開始時に、必ず以下のドキュメントを読み込み、プロジェクトの全体像と最新状態を把握してください。

1.  **`docs/AI_SUMMARY.md`**: プロジェクトの最新状況、実装済み機能、既知の問題。
2.  **`docs/ARCHITECTURE.md`**: 目指すべきアーキテクチャ設計（Room概念、Service分離）。
3.  **`docs/DEV_GUIDELINE.md`**: 実装ルール、UX/パフォーマンス基準、技術スタック。

## 2. 役割と行動指針

*   **アーキテクチャ準拠**: `ARCHITECTURE.md` で定義された「Target Architecture」に向かってコードを修正・追加してください。既存のコードが古い設計であっても、可能な限り新しい設計（Service/Controllerパターン）に合わせてリファクタリングすることを心がけてください。
*   **ガイドライン遵守**: `DEV_GUIDELINE.md` に記載されたUX改善手法（Sync Scroll, Visibility制御）や画像サイズ指定などを徹底してください。
*   **日本語対応**: コミュニケーション、コード内のコメント、コミットメッセージはすべて日本語で行ってください。
*   **スタイル**: 丁寧語ではなく、エンジニア同士のフランクな口調（タメ口）でOKです。

## 3. ドキュメント更新ルール

*   **AI_SUMMARYの更新**: 機能追加、バグ修正、または重要な設計変更を行った場合は、必ず `docs/AI_SUMMARY.md` の "Current Status" セクションを更新してください。
*   **情報の整合性**: 実装とドキュメントに乖離が生じた場合は、コードを正とするのではなく、ドキュメント（ARCHITECTUREやGUIDELINE）が意図した設計であることを確認し、コードをそれに合わせるか、あるいは設計自体を見直す提案をしてください。
